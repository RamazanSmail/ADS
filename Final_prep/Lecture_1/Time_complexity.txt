1. O(1)– Константная сложность.
Время выполнения не зависит от размера входных данных.
Пример: Доступ к элементу массива по индексу.

2.	O(log n)– Логарифмическая сложность.
Алгоритм сокращает объём данных вдвое на каждом шаге.
Пример: Binary search.

3.	O(n)– Линейная сложность.
Алгоритм проходит по всем элементам.
Пример: Finding the maximum in array.

4.	O(nlogn) – Квазилинейная сложность.
Обычно встречается в алгоритмах сортировки.
Пример: Быстрая сортировка (Quick Sort), Merge Sort, Heap Sort.

5.	O(n^2)– Квадратичная сложность.
Характерно для алгоритмов, использующих вложенные циклы.
Пример: Bubble Sort, проверка всех пар элементов.

6.	O(2^n) – Экспоненциальная сложность.
Время выполнения удваивается с каждым дополнительным элементом входных данных.
Пример: Решение задач перебором всех комбинаций (например, задача о рюкзаке).

7.	O(n!)– Факториальная сложность.
Алгоритмы, перебирающие все возможные перестановки.
Пример: Задача коммивояжёра (в простейшем случае).
